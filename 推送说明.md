# 🚀 快速推送指南

## 方法一：使用批处理脚本（最简单）

1. **双击运行** `推送到GitHub.bat`
2. 按照提示操作
3. 如果提示输入GitHub仓库地址，先创建仓库后输入地址

## 方法二：手动执行命令

### 步骤1：创建GitHub仓库

1. 访问 https://github.com
2. 点击右上角 "+" → "New repository"
3. 填写仓库名称（例如：`student-report-app`）
4. **重要**：选择 **Public**
5. **不要**勾选 "Initialize this repository with a README"
6. 点击 "Create repository"
7. **复制仓库地址**（例如：`https://github.com/你的用户名/仓库名.git`）

### 步骤2：执行推送命令

在项目文件夹中打开 PowerShell 或 CMD，执行：

```bash
# 初始化Git仓库（如果还没有）
git init

# 添加所有文件
git add .

# 提交代码
git commit -m "Initial commit: 学生报告展示应用"

# 添加远程仓库（替换为你的实际仓库地址）
git remote add origin https://github.com/你的用户名/仓库名.git

# 推送到GitHub
git branch -M main
git push -u origin main
```

### 步骤3：如果提示需要认证

如果提示输入用户名和密码：
1. 用户名：输入你的GitHub用户名
2. 密码：**不要输入密码**，而是使用 **Personal Access Token**

**如何获取Token：**
1. GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic)
2. 点击 "Generate new token"
3. 勾选 `repo` 权限
4. 生成后复制Token
5. 在提示输入密码时，粘贴Token

## 步骤4：启用GitHub Pages

1. 在GitHub仓库页面，点击 **Settings**
2. 找到左侧的 **Pages**
3. 在 "Source" 下选择：**GitHub Actions** ✅
4. 保存设置

## 步骤5：等待部署

1. 点击仓库顶部的 **Actions** 标签
2. 等待约 **2-3分钟**
3. 看到绿色的 ✅ 表示部署成功
4. 在 Settings → Pages 中可以看到网站地址

## 获取部署链接

部署完成后，你的网站地址：
```
https://你的用户名.github.io/仓库名/
```

## 使用部署链接

1. 打开应用，点击"接收数据"按钮
2. 输入你的GitHub Pages域名（例如：`username.github.io/repo-name`）
3. 生成分享链接
4. 手机可以直接打开！

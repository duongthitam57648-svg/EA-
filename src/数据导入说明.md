# 📥 学生数据导入功能说明

## ✅ 功能已实现！

现在您可以通过JSON格式导入学生数据到应用中。数据会自动填充到各个页面的对应位置。

## 🚀 如何使用

### 方式一：粘贴JSON数据
1. 点击左上角第二个数据库图标按钮（📊）
2. 在弹出的对话框中粘贴JSON数据
3. 点击"导入数据"按钮
4. 数据会自动填充到页面中

### 方式二：上传JSON文件
1. 点击左上角第二个数据库图标按钮（📊）
2. 点击"上传JSON文件"按钮
3. 选择您的.json文件
4. 点击"导入数据"按钮

### 方式三：使用模板
1. 点击左上角第二个数据库图标按钮（📊）
2. 点击"加载模板"按钮
3. 系统会自动加载示例数据
4. 您可以修改后再导入

## 📋 数据字段对照表

### 第一页 - 欢迎页
- `student_name` - 学生姓名（string）
- `current_level` - 当前级别（int）

### 第二页 - 学习统计页
- `days_since_first_large_order` - 首大单距今的天数（num）
- `current_class_hours` - 当前已上课时数（num）
- `current_learning_phase_completion_rate` - 当前学习阶段完成度（num）

### 第三页 - 数据分析页
- `exceed_student_percentage` - 当前学生超越的学生百分比（num）
- `preview_rate` - 预习率（num）
- `review_rate` - 复习率（num）
- `opening_rate` - 开口率（num）
- `highest_score_in_recent_unit_tests` - 近期单元测试最高分（num）

### 第四页 - 信封页
- `contents_of_the_letter_to_parents` - 给家长的信内容（string）
- `contents_of_the_letter_to_childs` - 给孩子的信内容（string）

### 第五页 - 科学成长页
- `current_english_growth` - 当前所在英语阶段（int, 1-5从下到上）

### 第七页 - 学习进度页
- `next_stage_of_language_skills` - 下一阶段语言技能和优势（string）
- `advice_after_class` - 课后建议（string）
- `current_ability_acquired` - 当前已获得能力（string）
- `school_ability` - 对应的学校能力（string）
- `current_classes_num` - 当前课程数量（int）
- `current_learning_progress_one` - 当前学习进度第一小块（string）
- `current_learning_progress_two` - 当前学习进度第二小块（string）
- `current_learning_progress_three` - 当前学习进度第三小块（string）

### 第八页 - 学习阶段对比页
- `current_level_corresponds_to_grade` - 当前级别对应年级（string）
- `current_learning_topics` - 当前学习主题（string）
- `target_level` - 目标级别（int）
- `target_level_corresponding_to_grade` - 目标级别对应年级（string）
- `target_learning_progress_one` - 目标学习进度第一小块（string）
- `target_learning_topics` - 目标学习主题（string）
- `target_ability_acquired` - 目标已获得能力（string）

### 第九页 - 视频播放页
- `preschool_video_link` - 学前视频链接（string）
- `post_learning_video_link` - 学后视频链接（string）

### 第十页 - 学习规划页
- `study_planning` - 学习规划（string）

## 📝 JSON 数据模板示例

请查看项目根目录的 `student-data-template.json` 文件，或者在导入对话框中点击"加载模板"按钮。

## ⚠️ 注意事项

1. **必需字段**：`student_name` 是必需字段，必须包含在JSON中
2. **数据类型**：请确保数字字段（num/int）使用数字类型，不要用引号括起来
3. **字段名称**：字段名称必须与文档中的完全一致（区分大小写）
4. **JSON格式**：确保JSON格式正确，可以使用在线JSON验证工具检查

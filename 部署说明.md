# 🚀 快速开始指南

## 情况一：本地开发和测试（不需要部署）

### 步骤1：安装依赖
```bash
npm install
```

### 步骤2：启动服务

**方式A：分别启动（推荐）**

打开两个终端窗口：

**终端1 - 启动前端：**
```bash
npm run dev
```
前端会在 `http://localhost:3000` 运行

**终端2 - 启动服务器：**
```bash
npm run server
```
服务器会在 `http://localhost:3001` 运行

**方式B：同时启动（需要先安装concurrently）**
```bash
npm install -g concurrently
npm run dev:all
```

### 步骤3：告诉后端同事

如果后端同事在**同一台电脑**或**同一局域网**，告诉他：
- **服务器地址**：`http://localhost:3001/api/receive-data`（本地）
- 或：`http://你的IP地址:3001/api/receive-data`（局域网）

查看你的IP地址：
- Windows: `ipconfig`（查找 IPv4 地址）
- Mac/Linux: `ifconfig` 或 `ip addr`

## 情况二：需要远程访问（需要部署）

如果后端同事在**不同的网络**，需要部署到服务器。

### 部署选项

#### 选项1：部署到云服务器（推荐）

1. **购买云服务器**（阿里云、腾讯云、AWS等）
2. **上传代码**
3. **安装Node.js**
4. **运行服务**

#### 选项2：使用内网穿透工具（临时测试）

- **ngrok**：`ngrok http 3001`
- **localtunnel**：`npx localtunnel --port 3001`
- **花生壳**：国内常用的内网穿透工具

#### 选项3：部署到Vercel/Netlify（仅前端）

前端可以部署到Vercel/Netlify，但服务器需要单独部署。

## 🔍 快速测试

### 测试服务器是否运行

打开浏览器访问：
- `http://localhost:3001/health` - 应该返回健康状态

### 测试POST接口

使用curl或Postman测试：
```bash
curl -X POST http://localhost:3001/api/receive-data \
  -H "Content-Type: application/json" \
  -d '{"student_name": "测试", "current_level": 1}'
```

## ❓ 常见问题

**Q: 后端同事说连接不上？**
A: 
- 检查服务器是否运行
- 检查防火墙是否阻止3001端口
- 确认IP地址是否正确
- 如果在不同网络，需要使用内网穿透或部署

**Q: 可以只部署前端吗？**
A: 不行，服务器必须运行才能接收POST请求。前端和服务器都需要运行。

**Q: 生产环境怎么部署？**
A: 
- 使用PM2管理Node.js进程
- 配置Nginx反向代理
- 使用Docker容器化部署
- 配置SSL证书（HTTPS）
